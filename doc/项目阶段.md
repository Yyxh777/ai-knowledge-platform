### **阶段 1：多文档支持与知识库扩展**（大约 2-3 周）

#### **目标**：

* 实现**单文档 RAG** 到 **多文档 RAG** 的扩展，支持多个文档来源（如技术文档、员工制度文档、项目文档等）。
* 学会如何将多个文档转化为向量，并存储在向量数据库中。

#### **步骤与实现**：

1. **文档收集与整理**：

    * 收集所有需要用于查询的文档（如技术文档、员工制度文档等）。
    * 通过工具（如 `PyMuPDF`、`pdfplumber`）提取 **文本内容**。

2. **文档向量化**：

    * 使用 **LangChain** 结合 **embedding 模型**（如 Sentence-BERT）将文档转换为 **向量**。
    * 将向量存储到 **向量数据库**（如 **Milvus** 或 **FAISS**）中，并且为每个文档存储其 **元数据**（如文件路径、文件类型、摘要等）。

3. **多文档 RAG 系统**：

    * 设计 **多个文档类别**（技术文档、员工制度、项目文档等），并为每个文档类型配置不同的 **Retriever**。
    * 使用 **LangChain** 结合 **RAG** 系统实现查询文档，并返回相应的 **文档内容**。

4. **查询接口**：

    * 使用 **LangChain** 创建查询接口，用户可以提出问题，系统会根据问题类型检索相应的文档并返回。
    * 通过测试不同类型的问题，确保 **RAG 查询** 能正确响应。

#### **学习与实践**：

* 学习如何使用 **LangChain** 实现文档加载、向量化和检索。
* 学习 **Milvus** 或 **FAISS** 向量数据库的基本使用，进行向量存储与检索。

#### **技术栈**：

* **LangChain**（文档加载、RAG 系统）
* **FAISS / Milvus**（向量数据库）
* **PyMuPDF / pdfplumber**（PDF 文档解析）

### **阶段 2：问题分类与查询决策**（大约 2-3 周）

#### **目标**：

* 实现 AI 判断是否需要查询文档、选择查询路径的功能。
* 通过 **LangGraph** 管理问题分类与查询决策流程。

#### **步骤与实现**：

1. **问题分类**：

    * 判断用户输入的问题是技术问题、制度问题，还是其他类型的问题。
    * 可以使用简单的 **关键词匹配** 或者 **文本分类模型** 来实现问题分类。

2. **查询路径决策**：

    * 根据问题分类，决定是否使用 **RAG** 查询文档。
    * 如果问题不涉及文档查询，直接返回答案（如通过大语言模型 GPT-3/4）。

3. **LangGraph 决策流设计**：

    * 使用 **LangGraph** 创建决策流，根据用户问题的类型选择正确的查询路径（使用 **RAG** 或直接回答）。

4. **优化查询**：

    * 基于用户问题的类别和目标，优化查询过程，确保返回的答案与问题高度匹配。
    * 引入 **少量例子（Few-shot）** 技术来提升模型的回答质量。

#### **学习与实践**：

* 学习 **LangGraph** 的基本用法，设计 **多步骤决策流程**。
* 理解如何结合 **LangChain** 进行问题分类和路径决策。

#### **技术栈**：

* **LangGraph**（流程决策管理）
* **LangChain**（问题分类与查询管理）
* **GPT-3/4**（非文档问题的解答）

### **阶段 3：角色感知与权限管理**（大约 2-3 周）

#### **目标**：

* 实现基于 **角色管理** 的权限控制，确保用户只能访问授权的数据。
* 集成 **bladeX** 的角色和权限管理系统，判断用户角色并返回适当的查询结果。

#### **步骤与实现**：

1. **用户角色与权限管理**：

    * 使用 **bladeX** 的权限管理模块获取当前用户的角色（如普通员工、项目成员、管理员等）。
    * 基于用户角色判断其能访问的文档范围。

2. **权限控制的决策流程**：

    * 使用 **LangGraph** 设计一个决策流程，在查询时首先判断用户的角色。
    * 根据角色决定是否可以查询特定的文档、项目信息等敏感数据。

3. **权限结合 RAG 查询**：

    * 在 **RAG 查询系统** 中加入权限判断，确保只有符合权限的用户才能查询敏感信息。
    * 管理员可以查看所有文档，普通员工只能查看部分文档。

4. **验证与测试**：

    * 创建不同角色的用户（如管理员、普通员工），测试权限控制是否有效。
    * 验证权限管理是否与文档查询正常集成。

#### **学习与实践**：

* 学习 **bladeX 权限管理**，了解如何管理用户角色和权限。
* 学习 **LangGraph** 中如何将权限判断与查询流程结合。

#### **技术栈**：

* **bladeX**（角色与权限管理）
* **LangGraph**（决策流与权限控制）
* **LangChain**（权限判断与查询集成）

### **阶段 4：LangGraph 流程设计与优化**（大约 3-4 周）

#### **目标**：

* 使用 **LangGraph** 设计复杂的决策流程，管理不同类型的查询路径。
* 提升系统的智能化和流程自动化能力。

#### **步骤与实现**：

1. **决策流程的设计与优化**：

    * 使用 **LangGraph** 设计多步骤决策流，决定是否调用 **RAG** 查询、是否需要 Tool Calling、如何处理权限判断等。
    * 优化查询流程，确保在复杂问题下能高效决策。

2. **权限控制与查询结合**：

    * 将 **LangGraph** 中的角色判断与查询过程结合，确保查询数据时严格按照权限控制。

3. **测试与调优**：

    * 进行多种类型的查询测试，优化 **LangGraph** 流程。
    * 确保系统在权限和查询决策方面能够高效、准确地执行。

#### **学习与实践**：

* 深入学习 **LangGraph**，掌握如何设计复杂的多节点决策流。
* 学习如何结合 **LangGraph** 与 **RAG 查询** 系统来实现更加智能化的系统。

#### **技术栈**：

* **LangGraph**（多步骤决策流管理）
* **LangChain**（查询管理与集成）
* **Tool Calling**（调用内部系统接口）

### **阶段 5：MCP（Model Context Protocol）与系统优化**（大约 4 周）

#### **目标**：

* 实现 **MCP** 标准，提升系统的扩展性和可维护性。
* 对系统进行优化，提升性能并确保其稳定性。

#### **步骤与实现**：

1. **MCP 标准化工具调用**：

    * 实现 **MCP**，规范化所有工具调用接口，为后续功能扩展打下基础。
    * 通过 MCP 使得所有工具调用都遵循相同的接口和标准。

2. **系统性能优化**：

    * 对文档检索、权限管理等模块进行性能调优，提升系统在大规模文档和高并发查询下的响应速度。

3. **日志与监控**：

    * 集成 **日志记录** 和 **监控系统**，方便在生产环境中监控系统运行状态，及时发现问题并进行修复。

4. **系统扩展与维护**：

    * 为未来可能的扩展（如更多的文档类型、更复杂的查询）做好准备，确保系统的灵活性和可扩展性。

#### **学习与实践**：

* 学习 **MCP 标准**，掌握如何设计符合标准的工具调用接口。
* 学习 **性能优化** 和 **监控集成**。

#### **技术栈**：

* **MCP（Model Context Protocol）**（标准化工具调用）
* **LangGraph**（决策流管理与扩展）
* **日志与监控**（系统维护与扩展）
