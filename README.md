

# **项目名称**：企业内部技术与制度 AI 助手（基于 bladeX 权限体系）

### **项目描述**：

该项目基于公司现有的 **bladeX 框架**，利用 **LangChain**、**LangGraph**、**RAG**、**Tool Calling** 和 **MCP**，构建了一个 **角色感知的企业内部 AI 助手**。系统能够提供技术文档、公司制度、项目信息等多类查询服务，并根据用户角色和权限进行动态决策，保证信息安全与合规。

---

### **功能描述**：

#### **1. 技术文档问答**（新人、实习生）

* **需求**：实习生或新人可以通过提问，自动检索并获取与 **bladeX** 框架相关的技术文档。

* **用到的技术栈**：

    * **LangChain**：用来构建检索增强生成（RAG）链路，实现从技术文档库中检索并回答用户问题。
    * **RAG**：通过嵌入和向量数据库来提高问题回答的准确性，避免模型的幻觉回答。
    * **向量数据库（Milvus/FAISS）**：存储技术文档的嵌入向量，支持快速检索。

* **亮点**：

    * 使用 **RAG** 和 **向量数据库** 来增强回答的准确性，避免 AI 出现幻觉，确保技术文档的权威性。
    * **LangChain** 提供灵活的文档检索与问答流。

---

#### **2. 公司制度查询**（所有员工）

* **需求**：员工可以查询公司政策、制度等内容。

* **用到的技术栈**：

    * **LangChain**：通过构建 RAG 系统，查询员工制度的相关条款。
    * **RAG**：确保公司制度文档的内容被准确提取并展示，避免人工处理的误差。
    * **向量数据库**：存储员工制度的向量化内容，以便快速检索。

* **亮点**：

    * 使用 **RAG** 提高答案的准确性，确保员工在查询公司政策时不会得到错误信息。
    * 可以通过版本控制确保制度内容的更新是准确的。

---

#### **3. 项目周期与项目信息查询**（权限感知）

* **需求**：员工可以查询公司正在进行的项目以及项目周期等信息。

* **用到的技术栈**：

    * **Tool Calling**：通过与公司内部系统（如项目管理系统）接口的集成，查询项目数据。
    * **LangGraph**：根据用户角色判断是否允许访问项目信息，支持多角色、权限控制的智能决策。
    * **MCP（Model Context Protocol）**：通过协议规范工具调用，确保信息访问权限的合规。

* **亮点**：

    * 通过 **LangGraph** 和 **权限控制** 确保不同角色只能访问到他们应有的项目信息，增加系统的安全性。
    * 使用 **Tool Calling** 与公司内部系统集成，数据来源更可靠。

---

#### **4. 通用技术问题解答**（技术团队成员）

* **需求**：技术团队成员可以提问公司内部的技术问题，AI 直接回答。

* **用到的技术栈**：

    * **大语言模型（GPT-3/4）**：回答通用技术问题，如数据库设计、编程语言等。
    * **LangChain**：对接外部 API 或数据库以提取必要的信息并生成回复。

* **亮点**：

    * 针对技术类问题，**GPT-3/4** 提供高质量的解答，而不需要复杂的检索过程。
    * 适配公司的内部知识体系，避免外部搜索干扰。

### **项目亮点**：

1. **角色感知与动态决策**：通过 **LangGraph** 和 **角色权限管理**，根据不同用户角色动态调整查询内容和数据访问权限。管理者可以访问更详细的项目信息，普通员工则受到访问限制。

2. **RAG 系统增强准确性**：使用 **RAG** 技术将外部文档和知识库结合，确保返回的信息是精确且权威的，避免了 AI 的幻觉问题。

3. **Tool Calling 集成**：通过与公司内部系统接口的集成，AI 能够调用真实的项目数据，并保证权限的安全性，支持自动化查询和操作。

4. **模块化架构**：该系统能够灵活地扩展新功能，如增加更多的技术问答库、集成新的企业系统等。

### **简历项目描述格式**：

#### **企业内部技术与制度 AI 助手（基于 bladeX 权限体系）**

**技术栈**： **LangChain**, **LangGraph**, **RAG**, **Tool Calling**, **MCP**, **向量数据库（Milvus/FAISS）**, **GPT-3/4**, **Spring Boot**, **bladeX 框架**

**项目描述**：
该项目基于公司现有的 **bladeX 框架** 和权限管理体系，构建了一个 **角色感知的企业内部 AI 助手**，能够通过 **LangChain** 和 **RAG** 技术，帮助员工高效查询技术文档、公司制度、项目信息等内容。同时，结合 **LangGraph** 决策流程和 **Tool Calling**，确保信息查询和操作过程中的权限安全。

**功能描述**：

* **技术文档查询**：通过 **RAG** 技术，实习生和新人能够快速查询到与 **bladeX 框架** 相关的技术解答。
* **公司制度查询**：所有员工可以根据权限访问公司制度，并通过 **RAG** 获取原文引用。
* **项目周期查询**：根据用户角色（普通员工、项目成员、管理者）不同，提供不同的项目信息，使用 **LangGraph** 和 **Tool Calling** 实现权限控制。
* **技术问题解答**：通过 **GPT-3/4** 提供高质量的技术问答服务，并结合 **LangChain** 提取公司内部知识库的技术解答。

**技术实现**：

* **RAG** 系统实现文档知识检索，结合 **Milvus/FAISS** 向量数据库提升文档检索速度和准确性。
* **Tool Calling** 技术集成公司内部系统，实现敏感数据的权限访问控制。
* **LangGraph** 用于角色感知决策流程，保证 AI 系统的智能化与安全性。

**项目亮点**：

* 高度集成 **RAG**、**LangChain** 和 **Tool Calling**，确保查询信息的安全性和高效性。
* **LangGraph** 提供灵活的智能决策流，能够动态调整查询流程。
* 完美融合了 **权限管理** 与 **AI 技术**，确保企业数据的安全访问。
