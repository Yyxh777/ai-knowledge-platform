## 二、所有值得写进简历的项目亮点

按"面试官最关心什么"来排序，分三档：

### 第一档：架构级亮点（写在项目描述里）

> **①  基于 LangGraph StateGraph 的有状态多节点 AI 决策流**
>
> 设计 DAG 工作流：权限预取 → 语义问题分类 → 多路 RAG 检索 → 流式答案生成。图结构可插拔、可回放，相比线性 Chain 具备更强的可扩展性与可观测性。

> **② RBAC 权限渗透式 RAG 检索**
>
> 权限管理不做在应用层过滤，而是将 Java 服务返回的 `access_levels` 数组翻译为 Milvus `filter` 表达式（`access_level in ["public","internal"]`），在向量检索阶段直接剪枝，权限错误"物理上"无法被检索到。`type` 和 `access_level` 字段均建立 BITMAP 索引，权限过滤不走全表扫描。

> **③ Redis 持久化会话记忆 + 有界 Reducer 防泄漏**
>
> 使用 `AsyncRedisSaver` 将完整 LangGraph State（含对话历史、权限上下文、检索结果）持久化到 Redis，实现多轮对话跨请求恢复；自定义 Reducer `_trim_messages` 限制历史上限为 20 条，保证 Redis 存储量恒定，防止长会话导致 payload 无限膨胀。

### 第二档：工程级亮点（写在技术细节里）

> **④ 真正的逐 Token 流式输出**
>
> 使用 `astream_events(version="v2")` 监听 `on_chat_model_stream` 事件逐 Token yield，WebSocket 实时推送，不等待整段答案生成完毕。区分 LLM 流式输出和固定字符串（无权限/无文档）两条路径分别处理。

> **⑤ 无权限与无文档的精确区分**
>
> RAG 检索为空时，自动执行无权限 filter 的二次查询：若二次有结果则返回"您没有权限查看该文档"，若仍无结果则返回"暂无相关文档"——两种语义不同，不混淆处理。

> **⑥ 模块级单例与进程级资源复用**
>
> LLM 客户端（classify/generate/direct 三个不同温度参数）、Embedding 模型、Milvus 客户端均以模块级单例初始化，所有并发请求共享，避免每次请求建立新连接。

> **⑦ thread_id 服务端生成，防止会话劫持**
>
> 格式 `{user_id}_{timestamp_ms}` 由服务端在 WebSocket 认证成功后生成并下发，客户端无法伪造，每次新连接创建独立会话。

> **⑧ fail-secure 权限降级**
>
> Token 无效、Java 服务超时、权限接口报错时，一律降级为最低权限 `["public"]`，不降级放行，符合企业安全规范。

### 第三档：细节级亮点（可在面试中主动提及）

- **PDF 解析用 PyMuPDF（fitz）而非 PyPDF2**：后者已停止维护，前者中文提取准确率更高，支持复杂排版
- **DOCX 同时提取表格内容**：`python-docx` 遍历 `table.rows` 提取结构化表格，而非只取段落文本
- **切片分隔符按语义层级排列**：优先在段落、句子边界切割，最后才按词语和字符切，减少语义割裂
- **批量向量化**：`embed_documents(chunks)` 一次 API 调用处理整篇文档所有切片，而非逐条调用
- **Milvus 集合幂等创建**：`has_collection()` 先检查，重启服务不会因集合已存在而报错
- **集中配置管理**：全项目唯一 `config.py` 入口，统一类型转换，杜绝各模块散落调用 `os.getenv`
- **中间件顺序设计**：CORS 在 AuthMiddleware 之前注册，确保 OPTIONS 预检请求不被认证拦截
